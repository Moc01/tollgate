# 系统架构

本文档描述 Tollgate 的系统架构。协议级细节见 [协议规范](./协议规范.md)。

## 系统总图

```
                                  ┌─────────────────────────┐
                                  │   AI 智能体应用          │
                                  │  （例：Curio、自定义）    │
                                  └────────────┬────────────┘
                                               │ 使用
                                               ▼
                                  ┌─────────────────────────┐
                                  │   @tollgate/agent       │
                                  │   - 包装 fetch          │
                                  │   - 处理 402            │
                                  │   - 签名支付            │
                                  └────────────┬────────────┘
                                               │
                                               │ HTTP 请求 (1)
                                               ▼
                                  ┌─────────────────────────┐
                                  │   付费 API 端点          │
                                  │   （任意 HTTP 服务）      │
                                  └────────────┬────────────┘
                                               │ 使用
                                               ▼
                                  ┌─────────────────────────┐
                                  │   @tollgate/middleware  │
                                  │   - 验证 access token   │
                                  │   - 缺失则返回 402       │
                                  └────────────┬────────────┘
                                               │
                       ┌───────────────────────┴───────────────────────┐
                       │ (2) HTTP 402 + 支付指令                        │
                       ▼                                               │
                  Agent 收到 402                                        │
                       │                                               │
                       │ (3) POST /v1/intent (challenge, agent_pubkey) │
                       ▼                                               │
   ┌─────────────────────────────────┐                                 │
   │   Tollgate 结算服务              │                                 │
   │  （Vercel Edge Functions）       │                                 │
   │   - 创建 payment intent          │                                 │
   │   - 验证链上交易                 │                                 │
   │   - 签发 access token (JWT)     │                                 │
   │   - 记录分析数据                 │                                 │
   └────────────────┬────────────────┘                                 │
                    │                                                  │
                    │ (4) 返回 Solana Pay URL + intent_id              │
                    ▼                                                  │
   Agent 构建 USDC 交易，用自己的钱包签名                                  │
                    │                                                  │
                    │ (5) 提交交易到 Solana                             │
                    ▼                                                  │
              ┌──────────────────┐                                     │
              │  Solana 主网      │                                     │
              │  （或 Devnet）    │                                     │
              └────────┬─────────┘                                     │
                       │                                               │
                       │ (6) Helius webhook 检测到 USDC 转账            │
                       ▼                                               │
              结算服务确认交易                                           │
                       │                                               │
                       │ (7) 签发 access token (JWT，短 TTL)            │
                       ▼                                               │
                  Agent 用 (8) `Authorization: Bearer <jwt>` 重试      │
                                                                       │
                       ┌───────────────────────────────────────────────┘
                       ▼
              中间件本地验证 JWT 签名
              （不需要再请求结算服务）
                       │
                       ▼
                  API 返回 200 OK
```

## 组件

### `@tollgate/shared`

基础包。纯 TypeScript 类型和工具，被所有其他包使用。不依赖 Solana 或框架运行时。

- **类型**：`PaymentIntent`、`AccessToken`、`EndpointConfig`、`RevenueSplit`、`Tollgate402Body` 等
- **JWT 工具**：sign、verify、decode（基于 `jose`）
- **Solana helper**：地址校验、USDC mint 常量、devnet/mainnet 检测
- **错误类**：`TollgateError`、`PaymentRequiredError`、`BudgetExceededError`

### `@tollgate/middleware`

服务端库。提供框架适配器（Express、Hono、Next.js Edge、原生 Node）。

```typescript
import { tollgate } from '@tollgate/middleware'

app.use('/search', tollgate({
  // 必填
  endpointId: 'wiki-search-v1',
  price: 0.001,                    // 单位 USDC
  recipient: 'BDqnQu...',          // Solana 地址（base58）

  // 可选
  splits: [                        // 替代 `recipient`；预校验和为 1
    { wallet: 'Alice...', share: 0.7 },
    { wallet: 'Bob...',   share: 0.2 },
    { wallet: 'Plat...',  share: 0.1 },
  ],
  tokenTtl: 300,                   // 秒；默认 600
  quotaCalls: 1,                   // 每个 token 允许的调用次数；默认 1
  freeTier: { dailyCalls: 100 },   // 可选：按 IP/agent_pubkey 的每日免费额度
}))
```

**热路径性能**：中间件本地验证 JWT 签名，公钥从结算服务的 `/v1/jwks` 一次性获取并缓存。**不**会每次请求都调用结算服务。

### `@tollgate/agent`

客户端库。包装 `fetch` 兼容的接口，调用代码无需修改。

```typescript
import { withTollgate } from '@tollgate/agent'

const fetch = withTollgate(globalThis.fetch, {
  wallet: kp,                      // Solana Keypair（devnet/mainnet）
  rpcUrl: 'https://...',           // Helius 或其他 RPC
  maxPricePerCall: 0.01,           // 预算约束，USDC
  maxTotalSpend: 1.0,              // 会话级总预算，USDC
  onPayment: (intent) => ...,      // 可选钩子（telemetry/UI）
})
```

包装器逻辑：

1. 上游调用一次
2. 200：立刻返回
3. 402 + 有效 `Tollgate-402` body：解析 intent，检查预算，签名并提交 USDC 转账，轮询结算服务获取 access token，用 `Authorization: Bearer <jwt>` 重试
4. 按 `endpointId` 缓存 access token，缓存到 token 过期

还提供 **LLM SDK tool 包装器**：

- `wrapAnthropicTools(tools, agentConfig)` — Claude tool use
- `wrapOpenAIFunctions(functions, agentConfig)` — OpenAI function calling
- `wrapMCPClient(client, agentConfig)` — Model Context Protocol 客户端

这些适配器拦截模型调用 tool 时发出的 fetch（当 tool 指向付费端点时）。

### 结算服务（`apps/settlement`）

无状态 HTTP 服务，部署到 Vercel Edge Functions。Endpoint 列表：

| 方法 | 路径 | 用途 |
|---|---|---|
| POST | `/v1/intent` | 创建支付 intent；返回 Solana Pay URL + `intent_id` |
| POST | `/v1/confirm` | Agent 在交易发出后调用；如果已确认则返回 access token |
| GET | `/v1/jwks` | JSON Web Key Set，供中间件验证 access token |
| POST | `/v1/webhook/helius` | 接收 USDC 转账的 webhook |
| POST | `/v1/endpoints` | 注册新 endpoint（认证：dashboard session） |
| GET | `/v1/analytics/:endpointId` | 时间序列收入数据（认证：dashboard session） |

**为什么用 Edge Functions**：结算对延迟敏感；Edge 在 agent 附近运行。状态存 Postgres（Supabase），读取在 edge 缓存。

### Dashboard（`apps/dashboard`）

Next.js 14 应用，给 API 提供方使用。通过 Privy 认证（邮箱/Google 登录自动创建 Solana 钱包）。

页面：

- `/` — 首页
- `/dashboard` — 总览：今日收入、累计收入、热门 endpoint
- `/endpoints` — 注册新 endpoint、修改定价/分账
- `/endpoints/[id]` — endpoint 详情：时间序列图、最近调用、热门调用方
- `/payouts` — 通过 MoonPay 提现到银行（MVP 后）
- `/docs` — 嵌入式文档站

### Curio（`apps/curio`）

Next.js 14 应用。演示产品。

页面：

- `/` — 搜索界面
- `/c/[id]` — 可分享的回答页（可审计）
- `/explorer` — 公开的最近回答 + 成本明细 feed

内部，Curio 使用 Anthropic Claude，配上一个让 agent 调用 5+ 个付费数据源的 tool 定义。Tool 调用被 `@tollgate/agent` 路由，自动支付每次调用。

### Examples（`examples/*`）

5 个小型 Hono 服务器，每个用 `@tollgate/middleware` 保护，提供假数据（或抓取的真实数据）。Curio 使用它们。

| Example | 模拟数据 | 价格 |
|---|---|---|
| `news-api` | 最新加密新闻标题 | $0.002 |
| `github-search` | GitHub 仓库搜索 | $0.001 |
| `wiki-api` | Wikipedia 摘要 | $0.0005 |
| `arxiv-api` | ArXiv 论文搜索 | $0.003 |
| `solana-docs-api` | Solana 文档问答 | $0.0005 |

## 数据存储

- **Postgres（Supabase）**：endpoints、payment intents、access tokens、调用日志、分账记录
- **Edge 缓存（Vercel KV / Upstash Redis）**：JWKS、最近 intent 查询、限流

详见 [数据模型](./数据模型.md)。

## RPC 策略

| 用途 | RPC 提供商 | 原因 |
|---|---|---|
| Agent 提交 USDC 转账 | Helius mainnet RPC | 可靠、快 |
| 监听结算服务的 USDC 入账 | Helius webhooks | 推送式，无需轮询 |
| Dashboard 读余额 | Helius RPC + Supabase 缓存 | RPC + 30 秒缓存 |
| Devnet 操作（测试/CI） | Helius devnet RPC | 免费层够用 |

## 故障模式与恢复

| 故障 | 检测 | 恢复 |
|---|---|---|
| Helius webhook 漏推一笔交易 | 对账 cron 每 60 秒重扫 Solana | 追溯签发 access token；agent 客户端轮询 `confirm` |
| Agent 交易确认但 agent 在重试前崩溃 | Token 短 TTL 但有缓存；agent 重启可领取 | 结算服务暴露 `GET /v1/intent/:id/token` 提供 5 分钟宽限 |
| API 提供方中间件拒绝有效 token | 本地 JWT 验证失败（如 JWKS 过期） | 每 60 秒刷新 JWKS；如果签名足够新则 fail-open |
| 支付期间网络分区 | Agent 等待结算超时 | 可配置超时 + 自动退款标记供运维审查 |

## 为什么用 JWT 而不是链上 NFT/PDA

两个原因：

1. **延迟**：每次请求做链上检查会加 ~400ms。JWT 验证只需微秒
2. **成本**：即使 Solana 手续费很低，每次 API 调用都做链上检查在规模化后也累加成本

JWT 由结算服务的私钥签名。中间件一次性获取 JWKS。信任根植于产生 JWT 的链上支付，但验证在链下。

如果某个部署需要完全链上验证（受监管环境），`middleware` 提供 `verifyMode: 'on-chain' | 'jwt'` 选项（MVP 后）。

## 安全边界

- **结算服务**是 access token 的唯一可信签发方。其私钥必须严格保护
- **中间件**对客户端不可信，但通过 JWKS 信任结算服务
- **Agent**信任自己的 keypair。它**不**信任中间件（验证 402 响应的签名）
- **402 响应在 v0.1 不签名**（待解决问题：恶意 endpoint 可能虚报价格）。v0.2 将用 endpoint 注册的 key 签名 402 body。v0.1 的缓解措施：agent 强制 `maxPricePerCall`

完整威胁模型见 [安全模型](./安全模型.md)。

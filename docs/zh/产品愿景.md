# 产品愿景

> *AI 智能体的自动售货机。*

## 一句话总结

**Tollgate** 是一个 Solana 原生的 HTTP 402 支付协议，让任何 API 都能向 AI 智能体按次收 USDC。它由两个 npm 包（服务端付费墙中间件 + 客户端自动支付 SDK）+ 一个结算服务 + 一个 dashboard 组成。

第一个用 Tollgate 搭建的应用是 **Curio**——一个搜索智能体，每次问答都实时调用多个付费数据源，把"AI 给 API 付钱"的全链路展示出来。

---

## 我们要解决的问题

智能体网络已经到来。AI 智能体在浏览、研究、交易、为用户做决策。但所有现存的支付通道都是为人类设计的：

- **Stripe** 要身份证、KYC、银行账户
- **信用卡** 假设有身份验证、月结账单、可拒付
- **PayPal/Venmo** 是人对人的，KYC 锁死
- **API key + 预付余额** 把智能体绑死在一个供应商，零可移植性

所以今天的"AI 智能体"用的是它人类主人的 API key 在付费。这破坏了所有有趣的智能体设计模式：

1. **多源编排**：一个想查 10 个付费 API 的智能体，需要 10 个 API key 都由人类预先配好
2. **长期自治**：一个想动态发现*新*付费 API 的智能体——做不到，没有人类介入它无法注册新供应商
3. **收入归属**：一个托管多智能体的平台，没有干净的方式知道哪个智能体在哪个 API 上消费了多少
4. **微观经济**：$0.001 的 API 调用 Stripe 处理不了（最低 $0.30 + 2.9%）

智能体网络需要一个不同的支付底层。

## 核心洞察

让这件事**只能在 Solana 上做**的五个特性：

| 属性 | 为什么重要 | 为什么是 Solana |
|---|---|---|
| **亚分手续费** | $0.001 的调用，手续费不能比调用本身贵 | Solana 手续费 ≈ $0.00025 |
| **亚秒级最终性** | 智能体一次会调上百个 API；支付不能阻塞 | Solana 出块 ~400ms |
| **无 KYC、无银行** | 智能体没有身份；支付必须是匿名的 | Solana 钱包是无许可的 |
| **原生 USDC** | 智能体需要价格稳定才能做预算规划 | Solana USDC 是所有 L1 中流动性最强的稳定币 |
| **可编程分账** | 多人维护的 API 需要自动、无需信任的分钱 | SPL token transfer 成本几乎为零 |

没有任何其他链同时满足这五点。以太坊 L2 在前两条接近，但 USDC 流动性碎片化、DEX 碎片化破坏稳定性。比特币闪电网络满足前两条但没有原生稳定币。中心化通道完全不满足第三条。

## HTTP 402——失踪的状态码

HTTP 在 1996 年就定义了 `402 Payment Required` 但从来没用过。这个 web 上失踪的状态码终于等到了它的应用——而 Solana 是正确的底层。

Tollgate-402 是 **HTTP 原生**的，所以它和现有所有基础设施（CDN、负载均衡器、网关、AI tool 定义）完全兼容。它不是新的协议层，它只是一个支付头。

## 三类用户

### 1. **API 提供方**——他们 install `@tollgate/middleware`

```typescript
import { tollgate } from '@tollgate/middleware'

app.use('/search', tollgate({
  price: 0.001,
  recipient: process.env.MY_SOLANA_WALLET,
}))
```

就这样。他们的 API 现在向任何调用方收 $0.001 USDC，Solana 上结算，不需要 Stripe 账号。

### 2. **AI 智能体开发者**——他们 install `@tollgate/agent`

```typescript
import { withTollgate } from '@tollgate/agent'

const fetch = withTollgate(globalThis.fetch, {
  wallet: agentWallet,        // 一个 Solana 密钥对，比如来自 Privy
  maxPricePerCall: 0.01,      // 预算约束
})

// 现在任何 402 响应都会被自动支付然后重试
const res = await fetch('https://wikipedia-pro.example.com/search?q=Solana')
```

他们的智能体现在透明地处理付费 API，不需要为每个供应商单独集成。

### 3. **平台**——Tollgate Cloud（免费层 + 高级 SaaS 功能）

API 提供方在 dashboard 上注册 endpoint、看实时收入、看哪些 agent 在调用、配置多人分账。可选：连 MoonPay 把收入提现到法币。

## 演示应用：Curio

**Curio** 是杀手级演示——一个完全建立在 Tollgate 上的 Perplexity 风格搜索智能体。当用户向 Curio 提问时，它：

1. 并发调用 5+ 个专业付费 API（新闻、github、wikipedia、arxiv、solana 文档）
2. 通过 `@tollgate/agent` 给每个 API 付 USDC
3. 用 Claude（也通过 Tollgate 付费）合成最终答案
4. 把答案返回给用户，附带每个数据源的成本明细，用户可以在 Solana Explorer 上审计

Curio 不只是一个 demo。它是黑客松后可以直接发布的真实产品，对平台和数据源运营方都有持续经济收益。

# 开发计划

11 天交付 Tollgate 黑客松提交版的执行计划。每个里程碑都有完成定义。

## 整体策略

我们围绕一个目标优化：**让一段 30 秒 demo 让 Colosseum 评委停下滑动**。

那段 demo 需要：
1. 一个真实的、已部署的付费 API
2. 一个真实的 AI 智能体在调用它
3. 真实的 USDC 在链上转移
4. 一个 dashboard 显示收入到账
5. 一个 Curio 答案，带成本明细，可审计地链接到链上交易

其他每个功能都为这个 demo、pitch video、或 technical demo 服务。

## 协作模式

- **Claude（AI）** 完成约 95% 的实现：设计、代码、文档、脚本、部署配置
- **人类（Moc01）** 负责：注册账号、申请 key、录视频、在另一台机器上做真实测试、在 Claude 提供的选项中做选择
- **沟通**：Claude 先写代码；人类审核；人类报告任何问题；Claude 调试

## 风险登记

| 风险 | 概率 | 影响 | 缓解 |
|---|---|---|---|
| Helius webhook 投递延迟 | 中 | 高 | 对账 job 每 60 秒重扫 Solana |
| Privy 嵌入式钱包配额 | 低 | 中 | 用 Privy 免费层；如被堵则回退到 Solana keypair |
| 评测期间 Vercel Edge 冷启动 | 低 | 中 | 录制前 5 分钟用 cron 预热 |
| 评测期间 Anthropic API 限流 | 低 | 高 | 备用 key 待命；代码内节流 |
| Devnet USDC 流动性 | 低 | 高 | 必要时自建测试 SPL token；明确文档说明 |
| 11 天截止超期 | 高 | 关键 | Phase 6-8 落后时激进砍功能 |

## 砍功能优先级（落后时按顺序砍）

1. PostHog/Sentry（仅留 console）
2. MoonPay 集成（跳过、标"敬请期待"）
3. Curio 公开 `/explorer` feed（仅留搜索）
4. Dashboard 的 splits 配置 UI（用 JSON textarea 替代）
5. OpenAI function 包装器（仅发布 Anthropic adapter）
6. Dashboard 移动响应（Curio + landing 必须响应；dashboard 可仅桌面端）

永不砍的：middleware、agent、settlement、Curio 搜索、链上支付。

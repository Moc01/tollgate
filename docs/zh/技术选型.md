# 技术选型

每个工具为什么选它，以及替代方案是什么。

## 语言与运行时

### TypeScript

- **为什么**：服务端和客户端共享类型。强推断、生态成熟
- **替代**：Rust（11 天太慢）、Python（agent 端 Solana 生态弱）

### Node.js 20+

- **为什么**：Vercel Edge runtime、Privy、Anthropic SDK 都需要它。18+ 原生支持 fetch
- **替代**：Bun。冷启动更快，但 2026 年部分 Solana 库不兼容

## Monorepo

### pnpm workspaces

- **为什么**：速度、磁盘占用、可靠性的最佳平衡。Vercel 部署原生支持
- **替代**：Turborepo（功能多但配置复杂；不需要它的 build cache）、Nx（过度设计）、Yarn workspaces（更慢）

### tsup（包构建）

- **为什么**：零配置 TypeScript bundler，输出 ESM 和 CJS，正确生成 `.d.ts`
- **替代**：直接 tsc（更慢，需要 ESM 后处理）

## Web 框架

### Next.js 14（App Router）

- **用在**：`apps/dashboard`、`apps/curio`
- **为什么**：server component 让我们在不分离 API 层的情况下混合服务端 Solana 逻辑和客户端 UI。Vercel 原生部署。App Router 给我们 Curio agent UI 的流式渲染
- **替代**：Remix（类似但生态更小）、SvelteKit（很好但团队不熟）、Astro（过度设计）

### Hono（付费示例 API）

- **用在**：`examples/*`
- **为什么**：30 KB、快、可在 Edge / Node / Bun / Cloudflare Workers 上运行。让我们演示 `@tollgate/middleware` 在每个 runtime 都能工作
- **替代**：Express（更重、仅 Node）、Fastify（类似 Hono 但仅 Node）

### Vercel Edge Functions

- **用在**：`apps/settlement`
- **为什么**：结算对延迟敏感（agent 等它返回）。Edge 在 100+ 区域运行，冷启动 < 50ms
- **替代**：Cloudflare Workers（类似但 Solana 库兼容性更差）、AWS Lambda@Edge（配置过多）

## UI

### shadcn/ui

- **为什么**：基于 Radix 和 Tailwind 的可复制组件。我们拥有源代码。完美匹配评委期待的 Linear/Stripe 美学
- **替代**：MUI（太固执）、Chakra（更重）、无样式 headless（11 天太慢）

### Tailwind CSS

- **为什么**：到达精致 UI 的最快路径。成熟
- **替代**：原生 CSS（迭代慢）、CSS-in-JS（server-component 兼容问题）

### Recharts

- **用在**：dashboard 分析
- **为什么**：声明式 React API，开箱即用即美观
- **替代**：Visx（更强但代码多）、Tremor（基于 Recharts）

### Framer Motion

- **用在**：Curio 智能体支付动画
- **为什么**："agent 付了 $0.001"的滚动条必须有生命感。Framer 是事实标准
- **替代**：react-spring（更底层）、CSS 动画（表现力差）

## Solana

### `@solana/web3.js` v2（模块化版）

- **为什么**：官方、维护良好
- **替代**：gill（Anza 的新模块化客户端）—— 有前途但不成熟；v0.2 再考虑

### `@solana/spl-token`

- **为什么**：USDC 是 SPL token；需要它的指令构造器
- **替代**：手写指令编码（11 天太易错）

### `@solana/pay`

- **为什么**：标准支付 URL 格式。钱包原生理解
- **替代**：自定义 URL 格式（会失去 Phantom 移动端兼容性）

### Helius RPC + Webhooks

- **为什么**：Solana 主网/devnet 上最可靠。Webhook 产品独一无二—— 推送式 USDC 入账通知，无需轮询
- **替代**：Triton One（裸 RPC 极佳，无 webhook 产品）、QuickNode（Solana 专注度差）、自建（11 天不可能）

## 认证

### Privy

- **用在**：`apps/dashboard`、`apps/curio`
- **为什么**：邮箱/Google 登录自动创建 Solana 钱包（embedded）。消除"你有钱包吗？"的摩擦。是黑客松的直接赞助商
- **替代**：Dynamic（类似）、Web3Auth（更重）、Phantom Connect（需要预装钱包；我们要零摩擦）

## 数据库

### Supabase Postgres

- **为什么**：托管 Postgres，行级安全（RLS），慷慨免费层，从 Edge runtime 通过 JS 客户端轻松连接
- **替代**：Neon（也很好，但我们偏好打包的 auth + storage）、PlanetScale（MySQL 对我们的 `jsonb` 用法不方便）

### Drizzle ORM

- **为什么**：类型安全、轻量、无 codegen 步骤，可在 Edge runtime 工作
- **替代**：Prisma（更重、edge-runtime 兼容性差）、kysely（仅 SQL builder，不够人性化）

## AI

### Anthropic Claude（`@anthropic-ai/sdk`）

- **用在**：Curio 的 agent loop
- **为什么**：2026 年 tool use 体验最好。Sonnet 4.6 快且足够便宜支持实时查询。Claude 是团队优势
- **替代**：OpenAI Responses API（能力相似；`@tollgate/agent` 的 OpenAI 适配器为兼容性而存在，但 Curio 主路径不用）

### OpenAI text-embedding-3-small

- **用在**：Curio 检索（需要相关性评分时）
- **为什么**：最便宜的高质量 embedding。1536 维
- **替代**：Voyage（质量更高但更贵）

## 加密

### `jose`

- **用在**：`@tollgate/shared` 的 JWT 签名/验证
- **为什么**：WebCrypto 原生、Edge runtime 兼容、原生支持 EdDSA
- **替代**：`jsonwebtoken`（仅 Node、无 EdDSA 除非用插件）

### `@noble/curves`

- **用在**：`web3.js` 过度的 Solana keypair 操作
- **为什么**：极简、经过审计、可 tree-shake

## 可观测性

### PostHog

- **为什么**：可自托管的开源产品分析 + 漏斗。我们想在 pitch 评估期间实时看到 demo flow 的转化
- **替代**：Amplitude（更精致但更贵）

### Sentry

- **为什么**：错误追踪。免费层覆盖黑客松

## 托管与 DNS

### Vercel

- **用在**：3 个 app + 结算服务
- **为什么**：零配置 Next.js + Edge Functions + 每个 PR 的 preview 部署。免费层很大

### Cloudflare DNS

- **为什么**：DNS-only 模式让 Vercel 处理其他一切。域名易迁移

## CI/CD

### GitHub Actions

- **为什么**：公开仓库免费、和 GitHub 原生集成
- **流程**：push 时 typecheck + lint + test；tag 时 build npm 包

### Changesets

- **为什么**：管理两个 npm 包的版本和 changelog。手动但可靠
- **替代**：semantic-release（更自动但更难控制）

## 开发工具

### Biome

- **为什么**：用一个工具替代 ESLint + Prettier，速度快 10×。2026 年成熟
- **替代**：ESLint + Prettier（仍然可以但更慢）

### Vitest

- **为什么**：快、原生 ESM、Jest API 兼容

## 明确不用的

- ❌ **Anchor / Rust 程序** — 我们不需要自定义链上程序。USDC SPL 转账 + JWT 信任就够了。节省 4-5 天
- ❌ **GraphQL** — 黑客松 API 表面过度设计
- ❌ **微服务** — 结算是一个服务、dashboard 是一个 app。不需要服务网格
- ❌ **Kubernetes** — Vercel 处理所有托管
- ❌ **自定义 L2 / 侧链** — 直接用 Solana 主网/devnet

## 中文版台词

### 0:00 – 0:30 — Repo 巡视

> *（VS Code 打开，文件树展开）*
>
> "Tollgate 是 pnpm monorepo。两个 npm 包，四个 app。
>
> *（指着每个）*
>
> `packages/shared` — 协议类型、JWT helper、Solana 工具。
> `packages/middleware` — 服务端付费墙。每个 endpoint 一行代码。
> `packages/agent` — 客户端 SDK，自动支付 402 响应。
> `apps/settlement` — access token 的可信签发方。
> `apps/examples` — Curio 用的 5 个付费 API。
> `apps/curio` — 演示产品，付费数据源版的 Perplexity。
>
> 跨 packages 64 个单元测试。每次推送跑 CI。许可证：MIT。"

### 0:30 – 1:00 — 代码 walkthrough

> *（打开 `packages/middleware/src/hono.ts`——展示有多短）*
>
> "这是服务端的全部集成。一个 Hono 中间件工厂。
>
> *（高亮 5 行）*
>
> ```ts
> app.use('/search', tollgate({
>   endpointId: 'wiki-search-v1',
>   price: 0.001,
>   recipient: process.env.MY_SOLANA_WALLET!,
> }))
> ```
>
> 压缩到 3 行也行。中间件检查 Authorization 头。缺失或无效？返回 402 + Tollgate 协议 body。"
>
> *（打开 `packages/agent/src/withTollgate.ts`）*
>
> "客户端，你包装 fetch。
>
> ```ts
> const fetch = withTollgate(globalThis.fetch, { wallet, rpcUrl, maxPricePerCall: 0.01 })
> ```
>
> 现在任何 402 响应都触发：解析 body，向结算服务创建支付 intent，构建并签名 USDC 转账，提交，轮询直到 access token 返回，用 token 重试原请求。对调用方代码完全透明。"

### 1:00 – 1:45 — 现场集成

> *（终端：空 Hono 项目）*
>
> "看着。我有个全新 Hono 服务器，没认证，开放访问。
>
> *（输入）*
>
> ```bash
> $ pnpm add @tollgate/middleware
> ```
>
> *（在 editor 打开 index.ts，加 middleware import + use 行）*
>
> "三行。保存。重启。"
>
> *（无认证 curl）*
>
> ```bash
> $ curl http://localhost:4001/api/search
> ```
>
> *（展示 402 + Tollgate-402 body 的 JSON）*
>
> "402 Payment Required。body 里有 `tollgate` 块：协议版本、结算 URL、endpoint id、价格、币种、网络、challenge。Challenge 就是 Solana Pay 的 reference。
>
> *（切到 node REPL 或一个用 @tollgate/agent 的脚本）*
>
> "客户端。我用一个之前充过钱的 devnet keypair。
>
> *（跑脚本通过 withTollgate 调 endpoint，tail 日志）*
>
> "看——agent 收到 402。POST intent 到结算服务。构建 USDC 交易。签名。提交到 Solana devnet。轮询 /v1/confirm。收到 JWT。带 Authorization 头重试。收到 200。
>
> *（在 Solana Explorer 上展示链上 tx）*
>
> "tx 在这里。400 毫秒确认。$0.001 USDC 已交付。"

### 1:45 – 2:15 — 架构

> *（打开 `docs/ARCHITECTURE.md` 展示系统图）*
>
> "两个信任边界。结算服务是唯一签发方。中间件用缓存 JWKS 本地验证 token——每次调用不需要往返结算服务。Agent 双方都不信任，但自己构建并签名支付。
>
> Helius webhook 把 USDC 入账推给结算服务，结算服务通过 RPC 独立再验证才签发 token。纵深防御。
>
> 每次调用延迟？结算大约 400 毫秒——Solana 最终性。Token 缓存后，TTL 内的后续调用完全跳过这一往返。"

### 2:15 – 2:30 — 后续

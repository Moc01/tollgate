# 配置指南

如何在本地跑起 Tollgate 并部署。

## 前置条件

- Node.js 20.10+ ([下载](https://nodejs.org/))
- pnpm 9+ (`npm install -g pnpm`)
- Solana CLI ([安装](https://docs.solana.com/cli/install-solana-cli-tools))
- Git

## 1. 克隆

```bash
git clone git@github.com:Moc01/tollgate.git
cd tollgate
```

## 2. 安装依赖

```bash
pnpm install
```

这会安装 monorepo 中所有 packages 和 apps。

## 3. API key 与环境变量

复制示例环境变量文件：

```bash
cp .env.example .env.local
```

填入以下 key：

| 变量 | 来源 | 用途 |
|---|---|---|
| `ANTHROPIC_API_KEY` | console.anthropic.com | Curio 的 AI agent |
| `PRIVY_APP_ID` | dashboard.privy.io | 嵌入式钱包认证 |
| `PRIVY_APP_SECRET` | dashboard.privy.io | 服务端 Privy 操作 |
| `HELIUS_API_KEY` | dev.helius.xyz | Solana RPC + webhooks |
| `HELIUS_WEBHOOK_SECRET` | dev.helius.xyz（创建 webhook 后） | webhook HMAC 验证 |
| `SUPABASE_URL` | supabase.com（项目设置） | 数据库 |
| `SUPABASE_SERVICE_ROLE_KEY` | supabase.com（项目设置 → API） | 结算服务写入 |
| `SUPABASE_ANON_KEY` | supabase.com（项目设置 → API） | 公开读取 |
| `TOLLGATE_JWT_PRIVATE_KEY` | 通过 `pnpm tg:keygen` 生成 | 结算服务签名 |
| `TOLLGATE_JWT_PUBLIC_KEY` | 与私钥同生成 | JWKS 发布 |
| `SOLANA_NETWORK` | `devnet` | 或 `mainnet-beta` 用于生产 |
| `USDC_MINT_ADDRESS` | `4zMMC9srt5Ri5X14GAgXhaHii3GnPAEERYPJgZJDncDU`（devnet） | 测试用 |

主网 USDC mint 是 `EPjFWdd5AufqSSqeM2qN1xzybapC8G4wEGGkZwyTDt1v`。

## 4. 生成 JWT 密钥

```bash
pnpm tg:keygen
```

这会写一对 Ed25519 公私钥到 `.env.local`（只跑一次）。

## 5. 设置 Solana Devnet 钱包

```bash
solana-keygen new --outfile ~/.config/solana/devnet.json
solana config set --url devnet --keypair ~/.config/solana/devnet.json
solana airdrop 2  # 拿到 2 SOL 用于测试
```

要拿测试 USDC：

```bash
pnpm tg:airdrop-usdc
```

这个脚本（在 `scripts/`）调用公开的 devnet USDC 水龙头给你的钱包打钱。

## 6. 设置数据库

在 supabase.com 创建一个项目。

```bash
pnpm db:generate     # 从 schema 生成迁移文件
pnpm db:migrate      # 应用到 Supabase
pnpm db:seed         # 可选：插入示例数据
```

## 7. 设置 Helius Webhook

部署 `apps/settlement` 之后：

1. 去 dev.helius.xyz → Webhooks → Create
2. URL：`https://<你的-settlement-部署>.vercel.app/api/v1/webhook/helius`
3. 跟踪的 token：USDC mint
4. 账户地址：留空（我们通过 Helius API 动态注册 endpoint 的收款地址）
5. 把 secret 复制到 `.env.local` 作为 `HELIUS_WEBHOOK_SECRET`

本地用 webhook 开发：用 [ngrok](https://ngrok.com) 或 [Cloudflared Tunnel](https://www.cloudflare.com/products/tunnel/) 暴露 `localhost:3001`。

## 8. 本地运行

在不同终端运行：

```bash
# 终端 1：结算服务
pnpm --filter settlement dev          # localhost:3001

# 终端 2：dashboard
pnpm --filter dashboard dev           # localhost:3000

# 终端 3：curio demo app
pnpm --filter curio dev               # localhost:3002

# 终端 4：示例付费 API
pnpm --filter "examples/*" dev:all    # localhost:4001-4005
```

或一次跑全部：

```bash
pnpm dev
```

## 9. 端到端测试

```bash
pnpm tg:e2e
```

这跑完整 E2E：
1. 启动一个付费示例 API
2. 用全新 devnet 钱包起一个 agent
3. 调用 API
4. 验证 402 → 支付 → token → 200 响应

## 10. 部署

```bash
# 结算
pnpm --filter settlement deploy

# Dashboard
pnpm --filter dashboard deploy

# Curio
pnpm --filter curio deploy

# 示例 API
pnpm --filter "examples/*" deploy
```

每个命令在 typecheck + build 之后跑 `vercel --prod`。

## 故障排查

### `pnpm install` 因原生构建失败

Windows 上安装 [Visual Studio C++ Build Tools](https://visualstudio.microsoft.com/visual-cpp-build-tools/) 重试。

### `solana airdrop` 失败

Devnet 限流。等 30 秒重试，或用 [Solana Faucet UI](https://faucet.solana.com/)。

### Helius webhook 不触发

最常见原因：收款账户没在跟踪。检查 dev.helius.xyz 的 webhook 配置。

### JWT 签名验证失败

可能是 JWKS 缓存。重启中间件 app 或等 60 秒。

### Privy 登录失败

确认 `PRIVY_APP_ID` 与 dashboard.privy.io 的项目 ID 一致。检查 Privy 允许的 origins 包含你的 dev URL。
